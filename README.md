🌍 GroundPulse

GroundPulse is an ESP32‑based subsurface human presence detection system that identifies **biological activity beneath debris** using **ground‑coupled seismic vibrations** supported by **acoustic confirmation**.

The system primarily detects **biomechanical vibrations** generated by living humans—such as breathing‑induced motion, cardiovascular micro‑vibrations, and involuntary body movements—that propagate through soil, concrete, and rubble.  
A microphone is used as a **secondary, conditional sensor** to listen for **intentional human responses** such as tapping or faint audible motion when rescue operations allow for silence.

GroundPulse performs real‑time frequency‑domain analysis (FFT), provides live feedback on an OLED display, and transmits detection data wirelessly using LoRa.  
It is designed as a **low‑cost, deploy‑and‑scan tool** for disaster response, search‑and‑rescue triage, and experimental underground life‑detection systems.

🧠 How It Works

GroundPulse detects **living human presence indirectly** by observing biological and intentional mechanical activity transmitted into the ground and surrounding debris.

The system follows a **priority‑based sensing strategy**:

### Primary Detection — Seismic (Always Active)
Seismic bio‑vibrations are the most reliable underground signals and are continuously monitored.

### Secondary Detection — Acoustic (User‑Controlled)
The microphone is enabled only when:
- Heavy machinery is paused
- Rescuers request a victim response check
- Silence can be enforced briefly

This mirrors real rescue protocols used in professional listening devices.

---

### Sensors & What They Detect

| Sensor | What it Detects |
|------|----------------|
| Piezo Disc + LM358 | Ground vibrations from breathing, heartbeat & involuntary motion |
| Microphone (e.g., MAX9814) | Intentional tapping, faint motion sounds, human response signals |
| ADXL345 (optional) | Environmental vibration reference & noise context |

---

📊 Signal Processing

- Seismic signals are sampled at **50 Hz**
- A **256‑point FFT** is computed in real time
- Analysis focuses on the **0.1–5 Hz biological band**
  - Breathing: ~0.2–2 Hz  
  - Heart‑related micro‑motion: ~1–3 Hz  

The microphone channel is **band‑pass filtered** and analyzed for **short‑duration, human‑like acoustic events**, not continuous audio.

---

📊 Detection Confidence Logic

### Score Composition

- Seismic frequency match (breathing / heartbeat) → **up to 55%**
- Temporal stability & FFT prominence → **up to 25%**
- Acoustic event detection (tapping / response) → **up to 10%**
- Cross‑sensor agreement & persistence → **up to 10%**

### Status Levels

- **0–30%** → Environmental noise  
- **30–60%** → Weak or uncertain signal  
- **60–80%** → Possible human presence ⚠️  
- **80–100%** → Strong human signature 🚨  

*Microphone input alone never triggers an alert.*

---

🖥 Hardware Used

- ESP32 DevKit V1 (ESP32‑WROOM‑32)
- Piezo vibration disc(s) with LM358 amplification
- MAX9814 / similar microphone module (conditional use)
- ADXL345 3‑axis accelerometer (optional)
- SSD1306 128×64 OLED display
- SX127x LoRa module (433 MHz)
- 18650 Li‑ion battery (solar‑charge capable)
- LED + Buzzer for alert indication

*Camera modules are intentionally excluded due to zero effectiveness under debris and high power consumption.*

---

🔌 Pin Connections

| Component | ESP32 Pin |
|--------|-----------|
| Piezo (ADC) | GPIO34 |
| Microphone (ADC) | GPIO32 |
| Battery Divider | GPIO35 |
| ADXL345 SDA / SCL | GPIO21 / GPIO22 |
| OLED SDA / SCL | GPIO21 / GPIO22 |
| LoRa SCK / MISO / MOSI | GPIO18 / GPIO19 / GPIO23 |
| LoRa NSS | GPIO5 |
| LoRa RST | GPIO14 |
| LoRa DIO0 | GPIO2 |
| Alert LED | GPIO4 |
| Buzzer | GPIO13 |

Battery sensing uses a **100 kΩ + 100 kΩ** voltage divider.

📡 LoRa Data Format GP,,,<seismic_energy>,<acoustic_event>,<human_flag>,<battery%>,<packet_id>

🖥 OLED Display Modes

### Normal Scan Mode
- Human Presence Confidence (%)
- Dominant biological frequency (Hz)
- Seismic activity level
- Acoustic monitor status (active / inactive)
- Battery voltage & percentage
- Scan status indicator

### Alert Mode
- Flashing border
- “HIGH PROBABILITY – VERIFY” message
- LED + buzzer activation
- Live detection metrics

---

🔐 Reliability & Safety

- ESP32 watchdog timer prevents system lockups
- Microphone ignored during high vibration conditions
- Invalid or saturated sensor readings automatically discarded
- Seismic detection always prioritized over acoustic input
- Battery voltage protected and continuously monitored
