# 🌍 GroundPulse

**GroundPulse** is an ESP32‑based **subsurface human presence detection system** designed to identify **biological activity beneath debris and rubble** using **ground‑coupled seismic vibrations**, supported by **conditional acoustic confirmation**.

The system focuses on detecting **biomechanical micro‑vibrations** generated by living humans—such as breathing‑induced motion, cardiovascular micro‑movements, and involuntary body dynamics—that propagate through soil, concrete, and collapsed structures.

A microphone is used strictly as a **secondary, operator‑controlled sensor** to listen for **intentional human responses** (e.g., tapping or light audible movement) when rescue conditions permit enforced silence.

GroundPulse performs real‑time signal analysis, presents live feedback on an OLED display, and transmits detection events wirelessly via **LoRa**.  
It is intended as a **low‑cost, battery‑powered, deploy‑and‑scan prototype** for disaster response research, search‑and‑rescue triage, and experimental underground life‑detection systems.

## 🧠 System Working Principle

GroundPulse detects **human presence indirectly** by observing **mechanical signatures of life** transmitted into surrounding ground and debris.

The system employs a **priority‑based sensing model**:

### Primary Detection — Seismic (Always Active)
- Ground vibrations are continuously monitored using **four parallel‑connected piezo discs**.
- Weak vibration signals are amplified using an **LM358 operational amplifier**.
- The ESP32 analyzes low‑frequency components related to **breathing and micro‑body motion**.

### Secondary Detection — Acoustic (Conditional)
- The microphone is enabled **only on operator request**.
- Used during quiet windows to detect **intentional response signals**.
- Acoustic data **never triggers alerts on its own**.

| Microphone Module | Intentional tapping / audible response detection |
| MPU6050 Accelerometer | Environmental vibration reference & motion context |

---

## 📊 Signal Processing Overview

- Seismic data sampled at low frequency  
- Analog conditioning performed before ADC  
- FFT applied to vibration data  
- Focused biological frequency band:
  - Breathing motion: ~0.2–2 Hz
  - Micro‑motion components: ~1–3 Hz

Acoustic signals are filtered for **short‑duration, human‑like events**, not continuous audio.

---

## 📈 Detection Confidence Logic

### Score Composition

- Seismic frequency signature match → **up to 55%**
- Signal stability & temporal persistence → **up to 25%**
- Acoustic response detection → **up to 10%**
- Cross‑sensor agreement → **up to 10%**

### Status Interpretation

- **0–30%** → Environmental noise  
- **30–60%** → Weak or uncertain activity  
- **60–80%** → Possible human presence ⚠️  
- **80–100%** → Strong human signature 🚨  

> Microphone input alone never generates alerts.

## 🖥 Hardware Used

- ESP32 DevKit V1 (ESP32‑WROOM‑32)
- Piezoelectric vibration discs ×4
- LM358 operational amplifier
- Microphone module (electret / amplifier‑based)
- MPU6050 MEMS accelerometer
- SSD1306 0.96″ OLED display
- LoRa Module S‑14432 (433 MHz)
- 18650 Li‑ion battery + holder
- TP4056 Li‑ion charging module
- DC‑DC boost converter (5 V output)
- LEDs and buzzer for alerts

---

## 🔋 Power Architecture

GroundPulse uses a **rechargeable Li‑ion power system**:

- **18650 Li‑ion battery**
- **TP4056 module** for charging and battery protection
- **Boost converter** to generate a stable **5 V rail**
- ESP32 powered via **VIN**
- 3.3 V rail used for logic‑level peripherals

This approach provides:
- High current capability (ESP32 + LoRa safe)
- Stable operation during transmission bursts
- Rechargeability for field deployment

## 🔐 Reliability & Safety Measures

- Watchdog‑protected execution
- Analog signal clamping and conditioning
- Acoustic sensing disabled during high vibration
- Invalid readings discarded automatically
- Stable Li‑ion power delivery with decoupling
